# Focus My Energy

## 核心理念:先看精力,再看日历

日历只知道你**几点有空**,却不知道那个空档适合做**什么样的**事。大多数人的默认做法是"下午三点有个空档,就把最难的方案排进去"——可三点往往正是精力低谷,硬啃只会低效又痛苦。

本 skill 把这件事反过来:**先识别用户一天中的精力窗口(高峰 / 低谷 / 回升),再按任务的认知负荷把它们配对进去。** 日历上已经定死的事(会议、接送孩子、面试)是不可移动的"锚点",我们围着锚点安排灵活工作,而不是让日历的空档决定该做什么类型的活。

这背后有几条被反复验证的规律,理解它们能让你排得更聪明,而不是照搬模板:

- **昼夜节律有形状**:多数人上午是**高峰**(适合分析、创作、决策),午后陷入**低谷**(适合行政、杂事),傍晚**回升**(适合协作、头脑风暴)。夜猫子整体后移,把这套节奏往后挪几个小时即可。
- **精力是波浪,不是直线**:专注力约每 90 分钟一个周期,连续深度工作后需要恢复。别把高峰窗口塞满到没有喘息。
- **深 vs 浅**:高认知的"深度工作"(写作、建模、策略、复杂决策)必须放在高精力时段;低认知的"浅层工作"(回邮件、报销、填表、例行会)在低谷也能做好——把它们从高峰挪走,等于给深度工作腾出黄金时间。
- **精力有限、需要回充**:高峰窗口通常只有当天最重要的**一件**深度任务能真正吃透。贪多会稀释产出。

## 工作流程

### 第一步:收集三样东西

排计划前,确认你手上有:

1. **任务清单**——用户要做的事。如果只给了模糊目标(如"推进项目 X"),帮他拆成具体可执行的任务。
2. **精力习惯**——用户几点最清醒、什么时候犯困。**如果用户没提**,别自己瞎猜,用一两个轻问题问清楚,比如:"你一般上午状态好还是下午?""几点最容易犯困?"——问完立刻排,不要变成问卷。若用户明确不想答,就按"上午高峰 / 午后低谷 / 傍晚回升"的大众节律来排,并说明这是默认假设。
3. **固定锚点**——已经定死、不能挪的事(会议、通勤、接送、饭点)。这些优先落位,其他任务围着它们排。

### 第二步:给任务标认知负荷

把每个任务归成三档(判断依据是"要动多少脑子、能不能被打断",而非任务大小):

- **🔴 高认知(深度)**:需要连续专注、创造性或复杂决策。写方案、编码、数据建模、战略思考、啃难文档、重要谈判。
- **🟡 中认知(协作/沟通)**:要投入但可被打断。1:1、评审会、回复重要邮件、整理思路、做 PPT。
- **🟢 低认知(行政/例行)**:机械、琐碎、几乎不费脑。报销、填表、订机票、归档、日常沟通、清收件箱。

### 第三步:把任务配进精力窗口

按这个优先级配对,并解释你为什么这么排:

- **高峰窗口 → 先放那件最重要的高认知任务**。保护它:这一段尽量不排会、不插浅层杂事。若高认知任务不止一件,把次要的挪到次高峰或第二天。
- **低谷窗口 → 批量处理低认知杂事**。把散落的行政小事攒到一起做,趁脑子不在状态时清掉。
- **回升窗口 → 放协作/沟通类**。傍晚适合开会、头脑风暴、社交型任务。
- **围着固定锚点排**:锚点前后留缓冲(尤其大会议后别紧接深度工作,人是累的)。
- **留白与回充**:每个深度块之后安排短暂休息;不要把窗口填到 100%,给意外和恢复留 20% 左右。

规划一周时,同样思路放大到"天"的粒度:把需要整块深度时间的大任务分配到干扰最少的日子,把会议和杂事聚到零散的日子,别让深度工作被会议切成碎片。

## 输出格式

用下面的结构输出,让用户一眼看清"什么时段做什么、为什么"。用 emoji 标精力档、用 `[标签]` 标认知负荷:

```
## 今日精力对齐计划

📌 固定安排(不可动)
  10:00–10:30  项目周会
  12:30–13:30  午饭

🔋 高精力窗口 (9:00–10:00 / 14:30–16:00)
  → 写产品方案初稿   [深度·高认知]  ← 今天最重要的一件事,优先保护
  → 数据建模分析     [深度·高认知]

😐 中等精力 (16:00–17:00)
  → 回复重要邮件     [协作·中认知]
  → 团队 1:1        [协作·中认知]

🪫 低精力 (17:00–17:30)
  → 报销 / 整理文件  [行政·低认知]

💡 为什么这样排
  - 把「产品方案」放在上午高峰、周会之前,趁头脑最清醒时啃硬骨头
  - 会议后不紧接深度工作,留了缓冲
  - 报销这类杂事攒到傍晚低谷统一清掉,不占用黄金时段
```

要点:
- **固定锚点单独列在最前**,让用户看到我们尊重了他日历上定死的事。
- **每个精力窗口下,任务按认知负荷从高到低排**,并标注 `[类型·认知强度]`。
- **高峰窗口里明确点出"今天最重要的一件事"**,并提示要保护它。
- **结尾用 3–5 条「为什么这样排」**说明关键取舍(为什么某任务放这、为什么某处留白),让用户能理解并微调,而不是盲从模板。
- 规划一周时,把顶层结构换成"周一 / 周二 …",每天再套上面的窗口逻辑;不必逐日全写满,重点标出深度工作日和杂事日的区分。

## 几条别忘的原则

- **忠实用户的节律,而不是套模板**:用户说自己是夜猫子、下午三点后才进入状态,就把高峰后移,别硬套"上午高峰"。
- **一天一块大石头**:高峰窗口只保一件真正重要的深度任务能吃透,别贪多。
- **杂事成批,不要零敲**:低认知任务分散着做最耗精力,攒起来在低谷一次清。
- **尊重锚点但不被锚点主宰**:固定的事围着安排,但灵活任务放哪由"精力是否匹配"决定,而不是"哪里恰好有空"。
- **信息不全就问,别硬编**:缺精力习惯时用一两个轻问题补齐;实在拿不到就用大众节律兜底并明说。
